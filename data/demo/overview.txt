Adaptive RAG is a retrieval-augmented generation design in which the system does not assume that every query needs the same retrieval behavior. A routing layer first decides whether a user question can be answered directly or whether external evidence should be retrieved. This avoids unnecessary retrieval for simple questions while preserving grounding for evidence-heavy ones.

A typical adaptive system may include three capabilities. First, query routing decides whether to skip retrieval, use a single retrieval step, or trigger a more advanced pipeline. Second, iterative retrieval allows the system to issue an initial search, draft an answer, identify remaining knowledge gaps, and launch follow-up retrieval. Third, self-reflection evaluates whether the answer is supported by the retrieved documents and whether the response should be revised, retried, or refused.
